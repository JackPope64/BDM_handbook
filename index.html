PUSHING TO GITHUB

First, make a repository at GitHub. Be sure to add a "READ ME" file.

Click the green "Code" button and copy the https:... address.

Open terminal in VSCode on the project. (ctrl+j)

In the terminal, enter git init

Check status if desired: git status

Enter git remote add origin https:... [the address you just copied from GH]

Enter git add .

Check status if desired: git status

Enter git commit -m "first commit"

Finally, enter git push -u origin master

Go back to the repo on GH, and refresh page.

There is a button on the left-hand side of the page, it should be labelled "main." Click on this and switch to "master." This will switch the project to the master branch.


PUSH PROJECT TO GITHUB PAGES

Click on "Settings."

Click on "pages" (left-hand side of page, toward the bottom)

The branch will be set to "none," click on that and choose "master" branch. Enter "save."

It will take about a minute or less. If desired, click on "code" (extreme left-hand side of page, near the top). Switch to master branch. Find the yellow dot. Click on that and it will show upload status. Otherwise just wait about 30 seconds and refrsh the page. There will be a link labelled "Your site is live at...." (or something similar). Click on it and you will see the page hosted on GitHub Pages.

Note: If you ever have to start over with the same project in a new repo, be sure to delete the .git folder in the project before continuing.

IMAGE LIBRARY

Here are some guidelines for collecting images from Pexels. Keep in mind we need to get royalty-free images, Pexels is good for that (as well as Unsplash, but I think they have gone to a paid model now, also their images may have watermarks which we don't want - so be on the lookout for those and avoid them.

Please go on Pexels to find images. Let's stay away from images with people in them, a few are ok, if they are small, people in the background, or nondescript, but I want mostly images without people. For sizes, download medium-sized images. I also prefer either .jps or .png files. Many will come over as .wepb or .avif which is ok, because we can convert them, but I'd like the majority to be either jpg or png.

When gathering images, put them in a (roughly) appropriate folder (for example, techy images should go in a folder called "tech," misc in a "misc" folder, etc)., then zip the folders and email them to me at contact@ByramDigitalMedia.com (my gmail account won't accept zipfolders).

I've included a list of suggested search terms. If you think of any others, feel free to include them.

Also feel free to suggest categories. I'm thinking of some now, like "House and Home," "Sports," "Art," "Music," "Nightlife," etc. We can really use anything, use lots of imagination (but let's keep them family-friendly!). I want to build up a nice-sized image library (a few thousand images) that we can draw from in the future.
